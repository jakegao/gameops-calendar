## 3. 产品方案设计

### 3.1 产品命名

**GameOps Calendar** — 游戏商业化运营排期中枢

### 3.2 产品定位

> 面向游戏商业化运营团队的**可视化排期管理协同平台**，以"运营日历"为核心，集成活动规划、跨团队协同、冲突检测、进度追踪和效果复盘于一体，让排期从"Excel表格"升级为"数据驱动的运营中枢"。

### 3.3 核心设计理念

```
                    ┌─────────────────┐
                    │   以"日历"为核心  │
                    │   所见即所排      │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
     ┌────────▼─────┐ ┌─────▼──────┐ ┌────▼────────┐
     │  多维度视图    │ │  智能协同   │ │  数据闭环    │
     │ 日/周/月/季   │ │ 实时同步   │ │ 排期→执行→  │
     │ 甘特图/看板   │ │ 变更通知   │ │ 效果→优化   │
     └──────────────┘ └────────────┘ └─────────────┘
```

### 3.4 产品核心流程

```
    ┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌─────────┐
    │  规划期  │ ──► │  排期期   │ ──► │  执行期  │ ──► │  上线期   │ ──► │  复盘期  │
    │         │     │          │     │          │     │          │     │         │
    │• 版本规划│     │• 活动创建 │     │• 任务分配│     │• 灰度发布│     │• 数据回收│
    │• 节点规划│     │• 时间排期 │     │• 进度追踪│     │• 全量上线│     │• 效果分析│
    │• 营收目标│     │• 冲突检测 │     │• 依赖管理│     │• 监控告警│     │• 经验沉淀│
    │• 竞品参考│     │• 团队同步 │     │• 风险预警│     │• 热修复  │     │• 模板优化│
    └─────────┘     └──────────┘     └──────────┘     └──────────┘     └─────────┘
```

## 4. 功能模块详解

### 4.1 模块一：运营日历主视图 🗓️

**核心功能：**

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **多维度日历视图** | 支持日/周/月/季/年五种时间粒度切换 | P0 |
| **多图层叠加** | 活动层、版本层、节假日层、竞品层可叠加/隐藏 | P0 |
| **拖拽排期** | 直接在日历上拖拽创建/调整活动时间 | P0 |
| **甘特图视图** | 以甘特图形式展示所有活动并行时间线 | P0 |
| **颜色编码** | 不同活动类型用不同颜色标识（限时活动/礼包/赛季/版本更新等） | P1 |
| **缩放浏览** | 支持鼠标滚轮缩放时间轴粒度 | P1 |
| **节假日/赛事标记** | 自动标注中国/全球节假日及电竞赛事日期 | P1 |
| **历史排期叠加** | 可叠加显示历史同期排期作为参考 | P2 |

### 4.2 模块二：活动排期管理 📋

**核心功能：**

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **活动模板库** | 预设常用活动类型模板（限时活动/赛季通行证/节日活动/促销礼包等） | P0 |
| **活动创建向导** | 引导式创建活动，自动填充关联信息 | P0 |
| **活动依赖设置** | 设置活动间前后置依赖关系 | P1 |
| **冲突检测引擎** | 自动检测时间冲突、资源冲突、同类型活动重叠 | P0 |
| **批量操作** | 支持批量创建、调整、复制活动排期 | P1 |
| **活动标签体系** | 多维标签分类（类型/优先级/负责模块/营收目标/状态） | P1 |

**活动类型分类体系：**

### 4.3 模块三：跨团队协同中心 🤝

**核心功能：**

| 功能 | 描述 | 优先级 |

| **角色视图过滤** | 各模块伙伴只看到与自己相关的排期任务 | P0 |
| **变更通知推送** | 排期变更自动推送通知给受影响的所有关联方 | P0 |
| **评审工作流** | 排期评审→修改→确认→锁定的流程管理 | P1 |
| **评论/批注** | 在排期项上直接评论讨论 | P1 |

| **导出能力** | 支持导出为PNG图片/PDF/Excel/iCalendar格式 | P0 |

### 4.4 模块四：智能辅助引擎 🧠

| 功能 | 描述 | 优先级 |
|------|------|--------|

## 文档总结

本方案基于对行业领先的游戏LiveOps管理工具（Metaplay、Meegle、AppMagic、Sensor Tower）和优秀开源项目（Strix GameOps、SVAR React Gantt、GSTC）的深度调研，设计了一款专为**腾讯游戏商业化运营经理**打造的排期管理工具 **GameOps Calendar**。

**核心价值主张：**
1. **所见即所排** —— 可视化日历/甘特图，告别Excel手工维护
2. **智能防冲突** —— 自动检测排期冲突，降低运营事故
3. **一键达全局** —— 排期变更实时同步所有合作模块
4. **数据驱动优化** —— 从排期到效果的完整闭环

**16周4个里程碑**的渐进式交付策略，确保每阶段都有可用成果，快速验证价值。
